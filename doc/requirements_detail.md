# 智码终端 (iCode Terminal) 详细需求文档

## 1. 产品概述
**智码终端 (iCode Terminal)**，简称 **iCode**，是部署在开发人员本地的命令行 (CLI) 安全工作入口。它基于开源项目 **OpenCode** 进行深度定制与增强，作为企业内网 Code LLM 的唯一合法客户端，负责“贴近代码现场”的风险管控与审计。

## 2. 核心功能需求

### 2.1 终端形态与标识 (CLI & Branding)
*   **单一形态**：现阶段仅提供 **CLI (Command Line Interface)** 形态，暂不开发 IDE 插件或浏览器扩展。
*   **品牌定制 (Rebranding)**：
    *   **启动 Banner**：移除 OpenCode 原有 ASCII Art，替换为 **iCode Terminal** 的定制 Banner。
    *   **命令别名**：主命令由 `opencode` 替换为 `icode`。
    *   **提示符 (Prompt)**：命令行交互提示符统一修改为 `icode> `。
    *   **版本信息**：`icode --version` 输出企业内部版本号及构建信息。
*   **身份绑定 (Identity Binding)**：
    *   **设备指纹**：采集终端硬件特征（MAC、硬盘序列号）生成唯一设备 ID。
    *   **多维鉴权**：支持“用户 + 设备 + 项目 + 仓库”的四维绑定。例如，User A 只能在 Company-Laptop 上访问 Payment-Repo。
    *   **身份认证机制**：现阶段主要依赖本地插件配置的 API Token 和设备指纹进行认证鉴权（暂不集成企业级 SSO）。

### 2.2 本地上下文采集与管控 (Context Awareness)
*   **基于 OpenCode 增强**：
    *   **复用 LSP 能力**：利用 OpenCode 原生的 LSP (Language Server Protocol) 支持，实现对本地代码的 AST 解析与引用查找。
    *   **智能上下文采集**：自动抓取当前编辑文件的 AST、光标位置代码、引用文件、最近修改的 Diff。
    *   **采集编译报错日志、测试失败报告、运行时堆栈信息。**
*   **最小必要上传 (Data Minimization)**：
    *   **.codeignore 机制**：支持类似 `.gitignore` 的配置文件，定义绝对禁止上传的文件（如 `config.yaml`, `.env`, `id_rsa`）。
    *   **自动裁剪**：对超长文件进行智能摘要，仅保留与 Prompt 相关的代码片段。
    *   **本地预检**：在请求发出前，计算上下文的 Token 消耗与敏感度分值。

### 2.3 本地敏感信息识别 (Local DLP)
*   **实时扫描**：在用户按下回车键（发送请求）的瞬间，对 Prompt 和 Context 进行毫秒级扫描。
*   **识别规则**：
    *   **凭证检测**：AWS AK/SK, GitHub Token, Private Key, JDBC Connection String。
    *   **隐私数据**：手机号、身份证、邮箱、内部员工工号。
    *   **企业机密**：识别内部项目代号（如 "Project-X"）、内网 IP 段 (10.x.x.x)、核心表名。
*   **处置动作**：
    *   **高亮警告**：在输入框内标红敏感词。
    *   **自动脱敏**：将 `password=123456` 替换为 `password=<MASKED>`。
    *   **阻断发送**：命中高危规则（如私钥）时，强制禁止请求发出。

### 2.4 用户交互与风险提示 (Human-in-the-loop)
*   **红绿灯机制**：
    *   对模型生成的代码进行风险分级：🟢 安全、🟡 需复核、🔴 高危。
    *   🔴 高危代码（如 `rm -rf /` 或硬编码密码）需用户手动输入“确认”才可插入编辑器。
*   **解释性提示**：
    *   当拦截发生时，告知用户：“您的输入包含‘生产数据库密码’，违反《信息安全规范》第 3.2 条，已自动拦截。”
*   **采纳率反馈**：
    *   记录用户是否采纳了 AI 建议（Tab 键接受 / 拒绝 / 修改后接受），用于评估模型质量。

### 2.5 本地行为审计 (Local Audit)
*   **全量行为记录**：
    *   记录“输入 Prompt”、“携带的 Context”、“模型返回 Code”、“用户最终修改后的 Code”。
    *   记录代码的**复制/粘贴**行为，防止用户绕过插件直接从外部网页复制不安全代码。
*   **离线缓存**：
    *   断网情况下，审计日志暂存本地加密数据库，网络恢复后自动上传至网关。

## 3. 非功能需求
*   **性能**：本地检测延迟 < 50ms，不造成 IDE 卡顿。
*   **兼容性**：支持 Windows 10/11, macOS (Intel/Silicon), Linux (Ubuntu/CentOS)。
*   **资源占用**：内存占用 < 500MB，CPU 占用 < 5%。

# 智码安全网关 (Smart Code Security Gateway) 详细需求文档

## 1. 产品概述
智码安全网关是企业内网 Code LLM 流量的统一枢纽。它接管所有终端发出的 AI 请求，进行集中式的鉴权、路由、清洗、审计和策略执行，并对接多种底层大模型（如 Qwen, DeepSeek, Llama）。

## 2. 核心功能需求

### 2.1 统一接入与路由 (Unified Access & Routing)
*   **多模型聚合**：后端对接私有化部署的开源大模型（如 Qwen, DeepSeek, Llama 等）以及商业模型 API。
*   **按安全等级智能路由 (Security-Tiered Routing)**：
    *   **高密项目隔离**：安全等级为“绝密/高密”的代码项目（如支付核心、风控算法），其所有 AI 请求强制路由至企业**内网完全私有化部署的 LLM**，确保数据绝对不出域。
    *   **普通项目分流**：安全等级为“普通/开源”的代码项目（如前端 UI、公共工具类），其请求可路由至性能更好、成本更低的云端商业模型 API 或共享大模型集群。
*   **流式处理 (Streaming)**：全链路支持 SSE (Server-Sent Events) 流式传输，确保打字机效果不卡顿。

### 2.2 统一身份与权限控制 (IAM & ACL)
*   **联合鉴权**：校验 `User Token + Device ID + Project ID` 的合法性。
*   **Git/SVN 权限对齐**：
    *   对接 GitLab/SVN 权限系统，确保 User A 无法通过 AI 问出他无权访问的 Project B 的代码。
*   **AI Agent 身份管理 (Agent Identity)**：
    *   **Agent ID**：为每个自动化 Agent 分配唯一身份。
    *   **MCP 管控**：发现并阻断未授权的 MCP (Model Context Protocol) 服务连接。
    *   **影子 AI 治理**：扫描内网中运行的未纳管 Agent。

### 2.3 输入/输出安全清洗 (Content Security)
85→*   **输入清洗 (Input Guard)**：
86→    *   **OmniGuard 级低延迟检测**：
87→        *   **极速响应**：在 LLM 推理链路中插入超低延迟检测层，确保即使对于长 Prompt 也能在 **20-100 毫秒** 内完成安全扫描。
89→        *   **流式介入 (Streaming Intervention)**：支持流式应用场景，在有害内容生成前实时阻断，而非等待生成完毕后再拦截。
90→    *   **Prompt 注入防御**：识别 "Ignore previous instructions" 等越狱攻击。
91→    *   **语义意图识别**：使用轻量级 NLP 模型识别恶意意图（如“帮我写个脚本扫描内网漏洞”）。
92→*   **输出清洗 (Output Guard)**：
93→    *   **越权内容阻断 (Access Control Enforcement)**：
94→        *   **核心痛点**：防止用户通过 AI 检索或生成其无权访问的项目代码（例如：普通开发者获取核心支付网关源码）。
95→        *   **上下文一致性校验**：实时比对用户权限（Git/SVN ACL）与 AI 生成内容所涉及的项目范围。一旦发现生成的代码片段属于用户无权访问的 `High-Sec-Repo`，立即熔断输出。
96→    *   **安全护栏模型 (Guardrail Model)**：部署专用小模型，对大模型生成的代码进行二次扫描。
    *   **漏洞检测**：识别 SQL 注入、XSS、命令注入、反序列化漏洞代码。
    *   **AI 代码溯源 (Code Provenance)**：
        *   **隐形水印**：借鉴 **Chalk** 技术，在返回的代码元数据中注入标记（Model Version, Request ID）。
        *   **AI-SBOM**：构建阶段生成 AI 代码物料清单，统计 AI 生成代码占比。

### 2.4 全链路可观测性 (Observability)
102→*   **Prism 风格控制台**：
103→    *   **实时流量监控面板**：顶部显示实时吞吐量（每分钟请求数 RPM）和拦截率。
104→    *   **事件流 (Event Stream)**：中间滚动展示每条拦截记录，包含：触发时间、检测类别、置信度分数、原始文本摘要（脱敏处理）。
105→    *   **筛选与下钻**：支持按检测类别、语言、时间段进行多维筛选。
106→    *   **Policy Editor**：提供可视化策略编辑器，无需编写代码即可配置自定义过滤规则。
107→*   **Agent 行为观测 (Geordie AI 风格)**：
    *   **Agent 仪表盘**：展示所有 Agent 的健康状态（绿/黄/红）。
    *   **Beam 引擎采集**：通过非侵入式探针，记录 Agent 的**思考链 (CoT)** 和 **工具调用序列**。
    *   **异常检测**：识别 **Inference Drift** (推理漂移) 和 **Tool Impersonation** (工具冒充)。
    *   **资产流向图谱**：可视化展示代码资产（Git Repo）流向了哪些模型。

### 2.5 审计与合规 (Audit & Compliance)
*   **全量留痕**：记录 Request/Response 全文（脱敏后）、耗时、Token 数、用户 IP、设备信息。
*   **会话回放**：像录像带一样回放高危会话的全过程。
*   **检测韧性 (Resilience)**：定期模拟攻击流量（BAS），验证拦截规则是否失效。

### 2.6 模型增强与知识库 (RAG & Fine-tuning)
*   **企业规范 RAG**：
    *   在 Prompt 中动态注入企业的《Java 开发规范》、《安全红线》。
    *   强制模型生成符合规范的代码（如“必须使用 Log4j2 而不是 System.out”）。
*   **测试反馈增强**：
    *   接收终端上传的测试失败日志，通过 Chain-of-Thought 推理，生成“错误归因 -> 修复建议 -> 验证代码”。

## 3. 非功能需求
*   **高可用**：支持 K8s 部署，多副本水平扩展，无单点故障。
*   **高并发**：单节点支持 500+ QPS 代码补全请求。
*   **数据安全**：所有落盘数据加密存储，敏感字段（如代码片段）单独加密。
